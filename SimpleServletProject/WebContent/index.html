Discussion:

What is the goal of using a deployment descriptor file by Java web applications?
ANSWER: it is a description about the web application context. so that we can configure web application context properties such as the servlet mapping and so on. it can possibly externalize the configuration.

What is the role of the two entries: servlet and servlet-mapping elements in web.xml file?
ANSWER: servlet - declare where and which Java servlet class file is. servlet-mapping - declare the web context URL mapping for the given servlet.

Discussion:

What is the difference between doGet() and doPost()?
ANSWER: doPost() require HTML form to be posted on the parameters to servlet. doGet() can process parameters directly from URL pattern and it expose the query. In contrast, doPost() can hide through using HTML form. E.g. the password field.

Activity 4: Initializing Servlets

Discussion:

Research on the init() and service() methods and understand their behaviour in the context of Servlets.
ANSWER:

  • init() override always get call at the first time servlet initialized. it is always before doPost(), doGet(), service() and any other potential servlet methods.
  • service() override depends of where we place our custom code before or after super.service() this will effect the before or after of doPost(), doGet() and potentially other HTTP verbs.
